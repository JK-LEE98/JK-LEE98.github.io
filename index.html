Real-time Notification System (SSE)

주문 및 결제 프로세스에서 발생하는 이벤트를 사용자에게 실시간으로 전달하기 위해
SSE(Server-Sent Events)를 활용한 알림 시스템을 구현했습니다.

  • 가맹점 → 본사 주문 요청 시 알림 전송
  • 본사 → 주문 상태 변경 시 알림 전송
  • 결재 반려 시 기안자에게 알림 전송 기능 구현
  • 알림 클릭 시 관련 페이지로 redirect 기능 구현
  • 읽은 알림 자동 삭제 기능 구현 (Scheduler)

Notification Architecture

비즈니스 로직에서 알림 이벤트가 발생하면 Notification Service가 알림을 생성하고
SseEmitter를 통해 클라이언트에게 이벤트를 전송합니다.
EmitterRepository로 사용자별 SSE 연결을 관리하고 Event Cache를 활용하여 알림 유실을 방지했습니다.


Notification Cache Management

알림 조회 성능을 개선하기 위해 Event Cache를 활용하여 알림 데이터를 관리했습니다.
알림 삭제 시 캐시 데이터도 함께 제거되도록 로직을 수정하여 캐시와 DB 간 데이터 불일치 문제가 발생하지 않도록 개선했습니다.


CQRS 구조

읽기와 쓰기 로직을 분리하기 위해 CQRS(Command Query Responsibility Segregation) 구조를 적용했습니다.
Command는 JPA로, Query는 MyBatis로 구현하여 역할을 명확히 나눴습니다.


Statistics Domain

서비스 운영을 위한 통계 데이터를 제공하기 위해 Statistics 도메인을 설계하고 구현했습니다.

실시간 주문 데이터를 직접 조회할 경우 조회 성능 저하와 원본 데이터 훼손 위험이 있기 때문에
별도의 통계 테이블을 생성하여 데이터를 관리하도록 구조를 설계했습니다.

Spring Scheduler를 활용하여 매월 지정된 시간에 이전 달의 주문 데이터를 집계하고
통계 테이블에 저장하도록 구현했습니다.

또한 사용자 역할(Role)에 따라 조회 가능한 통계 범위를 다르게 설계하여
가맹점주, 영업팀, 관리자 등 각 사용자 유형에 맞는 통계 데이터를 제공하도록 구현했습니다.


Statistics Scheduler

Spring Scheduler를 활용하여 매월 정해진 시간에 이전 달의 주문 데이터를 집계하고
통계 테이블에 저장하는 배치 작업을 구현했습니다.

통계 유형별 로직을 분리하기 위해 StatisticsGenerator 구조를 설계하여
새로운 통계 유형이 추가되더라도 쉽게 확장할 수 있도록 구현했습니다.

Troubleshooting

SSE 실시간 알림 Production 환경 문제 해결

Spring Boot / SSE / Concurrency / Notification

문제 상황

프랜차이즈 주문 및 결제 관리 서비스에서 SSE 기반 실시간 알림을 운영 환경에 배포했을 때,
연결은 유지되지만 이벤트 push가 바로 전달되지 않고 새로고침 후에야 도착하는 문제가 발생했습니다.

원인 분석

  • Emitter Map 순회 중 emitter 삭제가 일어나면서 ConcurrentModificationException이 발생할 수 있는 구조
  • 끊어진 SSE 연결이 Repository에 남아 있어 실제로는 죽은 emitter로 이벤트 전송을 시도하는 문제
  • Event Cache 삭제 시에도 Map 순회와 삭제가 동시에 일어날 수 있는 동시성 문제

해결 방법

  • Emitter Map을 직접 순회하지 않고, 복사본(Map copy)을 만들어 순회하도록 구조 변경
  • 이벤트 전송 중 IOException 발생 시 해당 emitter를 즉시 제거하여 끊어진 연결 정리
  • Event Cache 삭제 시 먼저 삭제 대상 key 목록을 수집한 뒤, 별도 단계에서 일괄 삭제하도록 로직 분리

결과 및 인사이트

  • Production 환경에서도 알림 이벤트 발생 시 클라이언트가 즉시 실시간 알림을 수신
  • SSE 연결 안정성 향상 및 끊어진 emitter 자동 정리
  • 컬렉션 순회 중 수정에서 발생할 수 있는 동시성 문제를 실전에서 경험하고 해결
